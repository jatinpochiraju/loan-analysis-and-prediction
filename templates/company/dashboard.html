Create Entity

Guidewire-style naming: new custom entities use a project prefix plus PascalCase name, for example `LSH_LoanProcess`. Existing base entities keep their base filename, and custom extension fields carry the prefix plus `_Ext` suffix.

Sample 1

Entity Name: LSH_LoanProcess

Supertype: PolicyPeriod

Subtype: LSH_LoanProcessExt

Sample 2

Entity Name: LSH_ClaimAudit

Supertype: KeyableBean

Subtype: LSH_ClaimAuditExt

Create Typelist/Typecode

Custom typelists should use the project prefix. Typecodes should be lowercase snake_case with the same prefix to avoid collisions with future base product additions.

Sample Typelist

Name: LSH_LoanApprovalStatusType

Initial Typecode: lsh_underwriting_review

Display Name: Underwriting Review

Extend Entity (EIX/ETX)

For OOTB entities, keep the entity filename/name unchanged and prefix only the custom field, for example `Policy.etx` with `LSH_DiscountCode_Ext`. Use `.eix` only for internal entity definition changes that cannot be handled in `.etx`.

EIX Sample

Entity: User

Field: LSH_InternalRouting_Ext

Type: string

Typelist: LSH_LoanApprovalStatusType

ETX Sample

Entity: Policy

Field: LSH_PriorityLevel_Ext

Type: entity_ref

Related Entity: LSH_VendorProfile
